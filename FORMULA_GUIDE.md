# 公式识别功能使用指南

## 功能概述

公式识别功能可以自动识别图片中的数学公式,并将其转换为LaTeX和MathML格式,方便在文档中使用。

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 运行应用

```bash
streamlit run src/app.py
```

### 3. 访问公式识别页面

- 点击侧边栏的"📐 公式识别"或首页的"公式识别"卡片
- 进入公式识别页面

## 使用方法

### 上传图片

1. 点击"📤 选择图片"按钮
2. 选择包含数学公式的图片文件
   - 支持格式: PNG, JPG, JPEG, BMP
   - 建议图片清晰度高,公式居中

### 识别公式

1. 上传后,点击"🚀 识别公式"按钮
2. 等待识别完成(首次使用需要下载模型,约500MB)
3. 查看识别结果

### 结果操作

- **查看LaTeX代码**: 在左侧文本框中显示识别的LaTeX代码
- **编辑LaTeX代码**: 可以直接修改识别错误的LaTeX代码
- **实时预览公式**: 右侧实时显示LaTeX代码的渲染效果
- **重置代码**: 点击"🔄 重置为原始识别结果"恢复原始代码
- **查看MathML代码**: 点击展开查看MathML代码(用于Word)
- **复制MathML代码**: 在MathML展开区域点击"复制"按钮
- **下载LaTeX文件**: 保存为 `.tex` 文件(使用编辑后的代码)
- **下载MathML文件**: 保存为 `.mathml` 文件(基于编辑后的代码)
- **下载Word文件**: 保存为 `.html` 文件,可导入Word使用
- **下载Markdown文件**: 保存为 `.md` 文件,包含代码和预览

### 历史记录

- 自动保存最近10条识别记录
- 可以查看历史识别结果
- 支持复制和删除单条记录
- 支持清空所有历史记录

### 验证和编辑识别结果

识别完成后，系统会提供以下验证和编辑功能：

**1. 左右分栏布局**
- 左侧：可编辑的LaTeX代码
- 右侧：实时的公式预览

**2. 实时预览**
- 修改左侧LaTeX代码时，右侧预览会自动更新
- 可以立即看到修改后的效果
- 如果LaTeX语法错误，会显示错误提示

**3. 识别错误修正**
- 如果识别的LaTeX代码有错误，可以直接在左侧编辑框修改
- 支持所有LaTeX语法操作
- 修改完成后，点击"下载"按钮会使用修改后的代码

**4. 重置功能**
- 如果修改后不满意，点击"🔄 重置为原始识别结果"
- 可以恢复到最初识别的结果

**5. MathML自动更新**
- 修改LaTeX代码后，MathML会自动重新生成
- 点击"查看MathML代码"展开可以看到更新后的MathML

**使用建议:**
1. 先查看右侧预览，判断识别是否正确
2. 如果有错误，在左侧编辑框修正
3. 实时查看右侧预览效果
4. 确认无误后，选择合适的格式导出

## 注意事项

### 首次使用

- 首次使用时会自动下载Pix2Tex模型(约500MB)
- 下载需要稳定的网络连接,请耐心等待
- 模型会缓存在本地,后续使用无需重新下载

### 图片质量要求

- **清晰度**: 图片越清晰,识别准确度越高
- **对比度**: 建议使用白底黑字的公式图片
- **公式位置**: 公式尽量居中,避免边缘被裁切
- **背景**: 避免复杂的背景图案

### 识别速度

- **CPU模式**: 单张图片识别时间约5-15秒
- **识别复杂度**: 复杂公式需要更长时间
- **系统配置**: CPU4核8G环境下性能良好

### 支持的公式类型

- ✅ 手写公式(清晰度要求较高)
- ✅ 印刷体公式
- ✅ 微积分公式
- ✅ 线性代数公式
- ✅ 希腊字母和数学符号
- ✅ 上下标、分数、根号等

### 不支持的情况

- ❌ 模糊不清的图片
- ❌ 背景过于复杂的图片
- ❌ 公式被遮挡或残缺
- ❌ 多个公式混在一起的图片

## 技术细节

### 使用的模型

- **Pix2Tex (LaTeX-OCR)**: 专门用于公式识别的深度学习模型
- 基于PyTorch实现
- 支持CPU和GPU推理

### 系统要求

- **Python**: 3.8+
- **内存**: 推荐4GB+
- **CPU**: 推荐4核+
- **磁盘**: 额外500MB用于模型缓存

### Docker部署

如果使用Docker部署,系统会自动处理:

1. 安装所有依赖
2. 配置资源限制(内存6GB, CPU 3.5核)
3. 持久化历史记录数据

部署命令:

```bash
docker-compose up -d
```

## 常见问题

### Q1: 识别失败怎么办?

A: 检查以下几点:
- 图片是否清晰
- 图片格式是否支持
- 网络连接是否正常(首次使用需要下载模型)
- 查看错误提示信息

### Q2: 识别速度慢?

A: 可能的原因:
- 首次使用下载模型中
- 图片较大或公式复杂
- 系统资源不足

### Q3: LaTeX代码错误?

A: 可能的情况:
- 识别模型生成的LaTeX可能有语法错误
- 可以手动修改LaTeX代码
- 复制到LaTeX编辑器中检查

### Q4: 如何提高识别准确度?

A: 建议:
- 使用高分辨率图片
- 确保公式清晰可见
- 避免背景干扰
- 使用印刷体而非手写体

### Q5: 如何在Word中使用识别的公式?

A: 方法:
1. 点击"复制MathML代码"或下载Word文件
2. 在Word中按Ctrl+V粘贴
3. 公式会自动转换为Word可编辑的公式格式
4. 可以像普通Word公式一样编辑

### Q6: LaTeX和MathML有什么区别?

A:
- **LaTeX**: 用于LaTeX文档、Markdown、Jupyter Notebook等学术环境
- **MathML**: 用于Microsoft Word、网页开发、支持MathML的文档编辑器
- 两者可以同时生成,根据使用场景选择合适的格式

### Q7: 历史记录会占用多少空间?

A: 
- 每条记录包含图片和元数据
- 平均每条约100-500KB
- 自动限制最多50条记录
- 总空间约25MB左右
